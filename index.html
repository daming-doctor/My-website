电商价格监控系统

实时监控电商平台价格变动，支持多商品阈值报警与日志管理

2025年7月
Python, PyQt5, Selenium, BeautifulSoup, SMTP
GitHub仓库

项目背景

亚马逊平台商品价格波动频繁，手动监控多个商品价格耗时且低效。本项目旨在开发一个桌面应用，帮助用户实时追踪多个商品价格，当价格低于设定阈值时自动发送邮件提醒，解决用户错失低价购买时机的问题。

实现过程

1. 界面设计：基于PyQt5构建直观的用户界面，包含商品列表、状态监控、功能按钮和配置区域，支持表格右键菜单快速操作。

2. 数据采集：结合Selenium和BeautifulSoup实现亚马逊商品信息爬取，支持动态内容加载和页面解析，提取商品标题、价格等核心信息。

3. 多线程处理：使用QThread实现多商品并行监控，避免界面卡顿，每个商品独立线程运行，互不干扰。

4. 状态管理：设计完整的状态机（初始化中→待执行→正在执行→完成并提醒/异常停止），清晰展示每个商品的监控状态。

5. 日志系统：为每个商品创建独立日志文件，记录价格检查历史、错误信息和状态变化，支持日志查看与清空。

核心功能

系统具备以下核心功能：

- 多商品管理：支持批量添加ASIN，自动初始化商品信息，显示标题、URL和当前状态
- 价格监控：可自定义监控频率（默认5秒/次），实时更新商品价格状态
- 阈值报警：当价格低于设定阈值时，通过SMTP发送邮件提醒
- 代理配置：支持代理IP设置，解决IP限制问题
- 日志管理：右键菜单快速查看/清除单个商品日志
- 状态追踪：记录成功/失败次数，异常状态自动标记

技术架构

系统架构：PyQt5界面层 → 多线程监控层 → 数据采集层 → 数据存储层 → 报警服务层

核心组件：
- 界面层：QWidget、QTableWidget构建用户界面，QMenu实现右键菜单
- 线程层：QThread实现NewTaskThread（初始化）和TaskThread（监控）
- 采集层：Selenium处理动态页面，BeautifulSoup解析HTML
- 存储层：JSON文件存储商品信息，文本文件记录日志
- 服务层：SMTP协议实现邮件报警，支持SSL加密连接

项目亮点

- 反爬策略：实现随机请求间隔、请求头伪装，降低被封禁风险
- 用户体验：表格右键菜单支持快速复制、查看日志等操作，提升效率
- 鲁棒性设计：完善的异常处理机制，网络错误自动重试，确保系统稳定运行
- 可扩展性：模块化设计便于添加新功能，如多平台支持、价格历史图表等

技术难点与解决方案

1. 动态页面解析：亚马逊部分页面使用JavaScript动态加载，采用Selenium模拟浏览器行为，确保价格数据正确提取。

2. 多线程同步：主线程与监控线程通过信号槽机制安全通信，避免UI刷新冲突，确保数据一致性。

3. 邮件报警稳定性：实现SMTP配置测试功能，支持SSL/TLS加密连接，确保报警邮件可靠送达。

4. 代理管理：支持多个代理IP轮询使用，自动跳过无效代理，提升爬虫抗封锁能力。

量化投资预处理分析报告优化

基于中证消费50指数的离群值处理与中性化技术优化方案

2025年6月
Python, 蒙特卡罗模拟, 稳健统计, 多因子模型
1000+次模拟测试

研究背景与意义

中证消费50指数聚焦食品饮料、家电、医药等核心消费龙头，但消费行业受宏观经济周期、政策、消费习惯变迁等多重因素影响，传统基本面分析难以捕捉动态风险收益特征。

该指数独特的窄基属性（仅50只成分股且集中于少数行业）导致其在量化建模中面临三大挑战：

  • 集群离群值敏感性问题
  • 行业偏态分布下的中性化适配矛盾
  • 预处理-因子检验链路失稳现象

本研究构建了基于蒙特卡罗模拟的定量评估框架，针对中证消费50指数多因子建模的预处理环节进行系统优化。

核心创新点

离群值处理方法创新

针对消费数据集群离群特性，在窄基指数场景验证MAD法相对Huber M估计的优越性：

  • 基于中位数估计的MAD法崩溃点高达50%
  • 在模拟行业事件冲击测试中，因子IC衰减率较Huber法低38%
  • 当离群值占比超15%时，MAD法信息损失率显著低于Huber法

中性化技术适配优化

突破宽基指数范式约束，解决行业偏态分布与小样本的双重挑战：

  • 通过引入行业变量与市值项，行业市值解释力提升至82.3%
  • 残余风险较指数内回归降低64%
  • 提出行业约束PCA方法，使正交因子经济可解释性从0.28提升至0.61

预处理-因子检验协同

验证预处理对下游分析的关键影响：

  • IC波动率降低22.6%
  • 分层收益反转频率下降41%
  • 因子收益率t值提升22.8%
  • 多空组合最大回撤降低36.2%

核心结论与建议

1. 区域集中效应明显：南山区作为深圳科技核心区，占据了近一半的数据分析师岗位，是求职者首选区域。福田区凭借金融优势位居第二，龙岗区则因制造业升级需求增长迅速。

2. 薪资结构呈金字塔型：初级岗位（＜12,000元）占比最大，但高级人才（＞25,000元）薪资溢价显著，反映出市场对资深数据分析师的高度重视。

3. 技能要求多元化：除技术能力（SQL/Python）外，业务洞察力和沟通能力成为关键软实力。数据可视化工具（Tableau/PowerBI）已成为行业标配。

4. 行业需求分化：互联网企业需求最大但竞争激烈，金融行业薪资最高但要求严格，零售电商行业提供最多初级岗位机会。

职业发展建议：求职者应夯实SQL/Python基础，掌握至少一种BI工具，积累行业知识，同时提升业务理解和沟通能力。初级从业者可关注零售电商行业积累经验，资深人士可瞄准金融和高科技企业。
